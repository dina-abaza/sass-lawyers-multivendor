# ⚖️ نظام إدارة مكاتب المحاماة (Lawyers Multivendor SaaS)

هذا المشروع عبارة عن نظام متكامل (SaaS) مبني بإطار عمل **Laravel** لإدارة مكاتب المحاماة متعدد المكاتب (Multivendor/Multi-tenant). يوفر النظام أدوات شاملة لإدارة القضايا، الجلسات، العملاء، الموارد البشرية، الحسابات المالية، والفواتير، حيث يتمتع كل مكتب (Tenant) ببيئته وبياناته المستقلة.

## ✨ المميزات الرئيسية (Key Features)

### 🏢 بيئة متعددة المكاتب (Multi-Tenancy)

- تسجيل مكاتب المحاماة كعناوين مستقلة.
- إدارة اشتراكات المكاتب (Tenant Subscriptions).
- فصل كامل للبيانات والملفات المرفقة لكل مكتب (باستخدام `tenant_asset`).

### ⚖️ الإدارة القانونية (Legal Management)

- **القضايا (Legal Cases):** إضافة القضايا، متابعة الحالات (أرشيف، متداولة، إلخ)، وربطها بالعملاء والمحامين.
- **الجلسات (Court Sessions):** جدولة جلسات المحاكم، التذكير بمواعيدها، وإرفاق مستنداتها.
- **الاستشارات (Consultations):** تصنيف الاستشارات (شفوية/مكتوبة، مدنية/جنائية..)، وتسجيل الردود عليها.
- **العقود (Contracts):** إبرام العقود مع العملاء ومتابعة تواريخ بدايتها وانتهائها.

### 👥 الموارد البشرية (HR & Attendance)

- **شجرة الموظفين:** إضافة الموظفين وتوزيعهم على الأقسام.
- **الحضور والانصراف (Attendance):** تسجيل الحضور جغرافياً (GPS/Radius) مع تحديد المواقع المسموحة (Work Locations).
- **المأموريات (Missions):** تسجيل المهام الخارجية للموظفين والمحامين.
- **الرواتب والخصومات:** إدارة مسيرات الرواتب (Salary Sheets)، وتطبيق الخصومات التلقائية (Deduction Sheets) عند الحضور خارج النطاق الجغرافي المحدد.

### 💰 النظام المحاسبي والمالي (Accounting & Invoicing)

- **شجرة الحسابات (Chart of Accounts):** نظام محاسبي مزدوج (دائن/مدين)، مع حسابات رئيسية وفرعية غير محدودة.
- **قيود اليومية (Journal Entries):** تسجيل القيود المحاسبية وضمان توازنها (المدين = الدائن).
- **كشف الحساب (Account Statements):** استخراج كشوفات حساب تفصيلية لأرصدة حساب معين بين فترات زمنية.
- **الفواتير (Invoices):** إصدار فواتير (شاملة الضريبة) تلقائياً للقضايا، الاستشارات، والعقود، مع تخصيص (شعار المكتب، الرقم الضريبي).

### 🛡️ الصلاحيات والأمان (Roles & Permissions)

- نظام صلاحيات متقدم للتحكم في الوصول.
- أدوار مخصصة تشمل: `Super Admin`, `Owner`, `Lawyer`, `Accountant`, `HR`, `User`.
- استعادة كلمة المرور عبر البريد الإلكتروني (OTP Check).

### 📂 إدارة المستندات (Document Management)

- رفع وأرشفة الملفات العامة للمكتب.
- إرفاق المستندات الخاصة بالعملاء، القضايا، والعقود بطريقة منظمة وآمنة.

## 🛠️ التقنيات المستخدمة (Tech Stack)

- **Backend Framework:** PHP, Laravel 11
- **Database:** MySQL / Relational DB
- **Authentication:** Laravel Sanctum (Token-based RESTful API)
- **Authorization:** Spatie Laravel Permission
- **Multi-Tenancy:** Architecture supporting Multi-tenant SaaS environments

## 🚀 طريقة التثبيت والتشغيل (Installation)

1. **استنساخ المشروع:**

    ```bash
    git clone <repository-url>
    cd lawyers-multivendor-main
    ```

2. **تثبيت الاعتماديات (Dependencies):**

    ```bash
    composer install
    ```

3. **إعداد ملف البيئة (Environment Variables):**

    ```bash
    cp .env.example .env
    php artisan key:generate
    ```

    _قم بتعديل بيانات الاتصال بقاعدة البيانات في ملف `.env`._

4. **تشغيل أوامر قواعد البيانات (Migrations & Seeders):**

    ```bash
    php artisan migrate --seed
    ```

5. **إنشاء رابط التخزين للملفات المرفقة (Storage Link):**

    ```bash
    php artisan storage:link
    ```

6. **تشغيل الخادم المحلي:**
    ```bash
    php artisan serve
    ```

## 📚 بنية وتصميم المشروع (Architecture)

تم بناء المشروع ليعمل كـ **RESTful API** لخدمة تطبيقات واجهات المستخدم (Frontend/Mobile)، حيث يتم تبادل جميع البيانات بصيغة `JSON`.

- **Controllers:** تحتوي على كافة العمليات المنطقية (CRUD) وجميعها تعيد كائنات و استجابات `JSON`.
- **Middleware:** تُستخدم للتحقق من الأدوار (Roles) والصلاحيات (Permissions) على مستوى كل وظيفة (Endpoint) لضمان أعلى معايير الأمان والفصل بين مسؤوليات أفراد المكتب الواحد.
